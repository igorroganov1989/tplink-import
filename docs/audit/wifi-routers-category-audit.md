## B.1. Что не так с текущими фильтрами в категории

### Краткий вывод

Фильтры в категории формально покрывают часть технических характеристик, но **плохо соответствуют сценариям выбора роутера**: есть дублирование с подкатегориями, бесполезные однозначные фильтры, пересекающиеся диапазоны скорости и отсутствие признаков, по которым пользователь реально отбирает товар (Mesh, 4G/LTE, портативные модели, скорость WAN/LAN). Часть значений заполнена неполно или неудобно для восприятия.

### Текущий состав фильтров (слева на странице)

| Фильтр | Значения |
|--------|----------|
| Цена | слайдер 0 – 20 299 ₽ |
| В наличии | один чекбокс |
| Стандарт WiFi | Wi‑Fi 7 / 6 / 5 / 4 (с IEEE) |
| Количество диапазонов | двухдиапазонный, однодиапазонный |
| DHCP‑сервер | есть |
| Кол‑во портов WAN | **только «1»** |
| Количество антенн | 1, 2, 3, 4, 6, **5** |
| Поддержка VPN | есть |
| Скорость WiFi | 150–1350 / >1350 / >3000 Мбит/с |
| Макс. скорость порта WAN | 10 / 100 / 1000 / 2500 Гбит/с |

---

### 1. Дублирование отбора по стандарту Wi‑Fi

На странице одновременно работают **два механизма** отбора по поколению Wi‑Fi:

- горизонтальные плитки-подразделы: «Роутеры Wi‑Fi 7 / 6 / 5 / 4» (ведут на `/catalog/routery-wi-fi/wi-fi-N/`);
- чекбоксы умного фильтра «Стандарт WiFi» с теми же значениями.

Пользователь не понимает, чем отличается клик по плитке от чекбокса. При переходе в подраздел (например, [Wi‑Fi 7](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-7/)) набор фильтров **сужается**, но плитки на родительской странице остаются — навигация и фильтр живут разными правилами.

![Обзор категории: плитки Wi‑Fi 7–4 и блок «Стандарт WiFi» в сайдбаре](./screenshots/01-category-overview.png)

**Рекомендация:** оставить один канал (либо подразделы без дублирующего свойства, либо только фильтр на общей странице).

---

### 2. Фильтры с единственным значением не помогают выбору

Ряд параметров не сужает выдачу:

- **«Кол‑во портов WAN»** — в фильтре только значение `1` (у типичных роутеров так и есть; отфильтровать «не‑1» нельзя).
- **«DHCP‑сервер»** и **«Поддержка VPN»** — только вариант «есть», без «нет» / «не указано».

Такие поля занимают место в сайдбаре, но **не отсекают ни одной позиции** и воспринимаются как шум.

![Панель фильтров: DHCP, WAN=1, VPN только «есть», антенны](./screenshots/03-filter-panel.png)

---

### 3. Некорректная сортировка и грубые диапазоны «Скорость WiFi»

**Количество антенн:** значения выводятся как `1, 2, 3, 4, 6, 5` — **«5» стоит после «6»**, что ломает ожидаемый числовой порядок (вероятно, сортировка как строк, а не как чисел).

**Скорость WiFi:** три варианта логически **пересекаются**:

- «150–1350 Мбит/с»
- «Больше 1350 Мбит/с» (включает всё, что выше 1350, в т.ч. >3000)
- «Больше 3000 Мбит/с» (подмножество предыдущего)

Пользователь не может однозначно выбрать сегмент «1350–3000» или «AX3000», а маркетинговые классы скорости из названий (AC1200, AX1500, AX3000) **не отражены** в фильтре.

![Фильтр «Скорость WiFi»: пересекающиеся диапазоны](./screenshots/05-wifi-speed-filter.png)

---

### 4. Нет фильтров под реальные сценарии покупки

В общей выдаче смешаны **разные классы устройств**, для которых нет отдельных фильтров:

| Что видно в листинге | Есть фильтр? |
|----------------------|--------------|
| Обычные домашние Wi‑Fi роутеры | частично (стандарт, диапазоны) |
| 4G/LTE (TL‑MR100, Archer MR500…) | **нет** |
| Портативные (TL‑MR3020, TL‑WR902AC) | **нет** |
| VDSL/ADSL (Archer VR400) | **нет** |
| Mesh / OneMesh (в названиях многих моделей) | **нет** |
| WAN/LAN 2,5 Гбит/с (AX55, AX80…) | только общий «Макс. скорость порта WAN», без привязки к количеству портов |
| Количество LAN‑портов | **нет** |

Покупатель, который ищет «роутер с SIM» или «mesh‑роутер», вынужден читать **все карточки**, а не пользоваться фильтром.

---

### 5. «В наличии» не совпадает с фактическими статусами карточек

В листинге встречаются статусы **«Достаточно» / «Мало» / «Под заказ»**, товары с ценой **0 ₽** (например, Archer MR550, Archer BE230 в подразделе Wi‑Fi 7). Фильтр даёт только бинарный чекбокс «В наличии» — неясно:

- попадают ли «Под заказ» в выдачу при включённом фильтре;
- скрываются ли позиции без цены;
- как соотносятся остатки склада и текст на карточке.

Это снижает доверие к фильтру наличия.

---

### 6. Несогласованность подразделов и умного фильтра

Подраздел [«Роутеры Wi‑Fi 7»](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-7/) содержит **2 модели**, обе «Под заказ», цена **0 ₽**. При этом на главной странице категории плитка Wi‑Fi 7 визуально равноправна с сегментами, где десятки SKU в наличии — **ожидание ассортимента не совпадает с реальностью**.

В подразделе Wi‑Fi 7 набор фильтров **урезан** (остаются стандарт, скорость, WAN) — поведение фильтра зависит от URL, а не только от выбранных чекбоксов.

---

### 7. Дубликаты и размытие SKU в выдаче (влияет на доверие к фильтрам)

На первой странице категории без применения фильтров:

- **TL‑WR842N** встречается **дважды** (разные URL: общий и `/wi-fi-4/…`);
- **Archer AX80** — два оффера **(EU)** и **(RU)** с одним артикулом `Archer AX80`.

Умный фильтр считает элементы инфоблока, а не «модель для покупателя» — пользователь видит **раздутую выдачу**, счётчики в фильтре (`Показать N товаров`) не отражают число уникальных моделей.

---

### 8. Прочие замечания по UX фильтра

- **Сортировка «По умолчанию (возрастание)»** — неочевидный критерий (не цена и не популярность).
- Длинная панель фильтров на мобильных уходит в **нижний шторный блок**; cookie‑баннер перекрывает часть опций (см. полный скрин страницы).
- Код свойства в разметке: `max_skorost_wlan` при подписи «Максимальная скорость **порта WAN**» — признак путаницы при заведении свойств (риск ошибок при сопровождении).
- Режим **«таблицей»** не даёт удобной матрицы характеристик для сравнения моделей по Wi‑Fi/WAN/LAN — только иной вид списка карточек.

## B.2. Что не так с характеристиками карточек в этой же категории

### Краткий вывод

Вкладка «Характеристики» заполнена **неравномерно**: у массовых моделей (Archer C6, AX55) — 40–45 свойств, у Wi‑Fi 7 (BE230) — **5 полей**. Повторяющиеся проблемы: числа **без единиц измерения**, противоречия между `<title>`, H1 и спеками, **ошибочная группа «ADSL»** у Ethernet‑роутеров, разный набор полей у моделей одного класса, **дубликаты SKU** с расхождением габаритов при одном артикуле.

Источник данных: вкладка **«Характеристики»** (`#props`) на карточках товаров, проверка 09.07.2026.

---

### Пример 1. Почти пустая карточка Wi‑Fi 7

**Карточка:** [Archer BE230](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-7/archer-be230/)  
**Проблема:** пустые / отсутствующие ключевые поля.

В H1 заявлены BE3600, двухдиапазонный Wi‑Fi 7, в описании — OFDMA, MU‑MIMO, порты 2,5 Гбит/с. Во вкладке «Характеристики» заполнено **только 5 свойств**:

| Свойство | Значение |
|----------|----------|
| Модель | Archer BE230 |
| Стандарт WiFi | Wi‑Fi 7 (802.11be) |
| Комплект поставки | Wi‑Fi роутер Адаптер питания Кабель Ethernet RJ45 Краткое руководство по настройке |
| Страна производства | Китай |
| Сертификация | CE, FCC, RoHS |

**Отсутствуют:** скорости по диапазонам, количество антенн, порты WAN/LAN (в т.ч. 2,5 Гбит/с), Mesh, WPA3/HomeShield, размеры, вес, гарантия, процессор, ОЗУ. Цена на витрине — **0 ₽**, статус «Под заказ».

![Archer BE230: вкладка «Характеристики» почти пустая](./screenshots/b2-be230-empty-specs.png)

### Пример 2. Противоречие AC750 / AC1300 на одной карточке

**Карточка:** [Archer C6](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-5/archer-c6/)  
**Проблема:** несогласованные значения в разных блоках страницы.

| Источник | Заявленный класс скорости |
|----------|---------------------------|
| `<title>` страницы | **AC750** («роутер Wi Fi AC750») |
| H1 на карточке | **AC1300** |
| Свойство `Скорость 802.11n, 2.4 ГГц` | 300 |
| Свойство `Скорость 802.11ac, 5 ГГц` | 867 |
| Сумма по спекам | **1167 Мбит/с** (не 750 и не 1300) |

Покупатель видит три разных «класса» скорости на одной странице. Это указывает на рассинхрон SEO‑заголовка, маркетингового H1 и импортированных характеристик.

**Проблема:** одно и то же свойство выводится как голое число — нельзя однозначно сравнить модели и выгрузить в маркетплейсы.

| Карточка | Свойство | Значение на сайте | Ожидаемый формат |
|----------|----------|-------------------|------------------|
| [Archer C6](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-5/archer-c6/) | `Частота процессора` | `1200` | 1200 **МГц** |
| [TL-WR842N](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-4/tl-wr842n/) | `Частота процессора` | `580` | 580 **МГц** |

### Пример 4. Секция «ADSL» у роутеров без ADSL

**Карточки:** [Archer C6](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-5/archer-c6/), [TL-WR842N](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-4/tl-wr842n/)  
**Проблема:** одно свойство под вводящим в заблуждение заголовком группы.

В обеих карточках в группе **`ADSL`** (не «Wi‑Fi» / «Радиопокрытие») выводятся:

- `Дальность беспроводной связи, мин.` = 50  
- `Дальность беспроводной связи, макс.` = 70  

У C6 — гигабитный Ethernet‑роутер; у WR842N — N300 + USB 3G/4G, **без ADSL/VDSL‑модема**. Название группы наследовано от шаблона и не соответствует типу устройства.

![TL-WR842N: группа «ADSL» с дальностью Wi‑Fi](./screenshots/b2-wr842n-adsl-section.png)

### Пример 5. Ключевая характеристика из H1 отсутствует в спеках

**Карточка:** [Archer AX55](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-6/archer-ax55/)  
**Проблема:** пустое / незаполненное поле для заявленной в заголовке функции.

- **H1:** «порт WAN/LAN **2,5 Гбит/с** и поддержкой Mesh»
- **В спеках, блок «Порты»:** только `Количество выходных портов 10/100/1000BASE-TX` = 4, `Кол-во портов USB 3.0` = 1
- **Нет** свойства вроде `Количество выходных портов 100/1000/2500BASE-T` (как у [Archer AX80(EU)](https://tp-link.ru/catalog/routery-wi-fi/archer-ax80-eu/))

Порт 2,5 Гбит/с — главный дифференциатор модели в листинге, но в таблице характеристик **не зафиксирован**, поэтому фильтр «Максимальная скорость порта WAN» и сравнение с AX80 работают некорректно.

### Пример 6. Виджет «Подходит ли Вам» противоречит характеристикам

**Карточка:** [TL-MR100](https://tp-link.ru/catalog/routery-wi-fi/tl-mr100/)  
**Проблема:** разные названия и значения одного и того же смысла («сколько устройств выдержит»).

| Блок | Текст |
|------|-------|
| Буллет в шапке | «Доступ к 4G LTE для **32 Wi‑Fi устройств**» |
| Свойство `Преимущества` | «…беспроводное 3G/4G подключение для **32 устройств** одновременно…» |
| Виджет `Число устройств` | **«до 1»** |

Дополнительно: у MR100 **нет** свойства `Стандарт WiFi` (есть только legacy‑поля `Стандарт Wi-Fi 802.11b/g/n`), тогда как у соседних моделей используется единое поле `Стандарт WiFi` = `Wi‑Fi 4/5/6…` — **один признак под разными названиями** в разных карточках.

![TL-MR100: виджет «до 1 устройства» при заявленных 32 в описании](./screenshots/b2-mr100-widget-conflict.png)

### Пример 7. Placeholder-текст вместо характеристики покрытия

**Карточка:** [TL-WR842N](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-4/tl-wr842n/)  
**Проблема:** незаполненное поле / техничеческий черновик на витрине.

В блоке «Подходит ли Вам TL-WR842N?» свойство `Площадь покрытия` содержит текст:

> «то же устройство для карточки товара в разделе роутеров: Wi-Fi N300 + поддержка 3G/4G USB-модема»

Это служебная заглушка, а не площадь в м². Рядом `Число устройств` = «до 1» — тоже не согласуется с классом устройства (4 LAN‑порта, VPN, USB).

### Пример 8. Один артикул — разные габариты (дубликаты SKU)

**Карточки:** [Archer AX80(EU)](https://tp-link.ru/catalog/routery-wi-fi/archer-ax80-eu/) и [Archer AX80(RU)](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-6/archer-ax80-ru/)  
**Проблема:** несогласованные значения при одном артикуле.

| Свойство | AX80(EU) | AX80(RU) |
|----------|----------|----------|
| Артикул на витрине | Archer AX80 | Archer AX80 |
| `Размеры (Ш × Д × В)` | 200 × 189 × 59 мм | **277,3 × 183,9 × 57,3 мм** |
| `Сертификация` | CE, RoHS, JPA, JRF, VCCI | **FCC**, RoHS |
| `Модель` | Archer AX80 | Archer AX80 |

Скорости Wi‑Fi совпадают (1148 + 4804 Мбит/с), но корпус и сертификация различаются — покупатель не может понять, какая ревизия придёт по артикулу `Archer AX80`.

### Пример 9. Разный полнотный набор полей в одном классе (AX55 vs C6)

**Проблема:** отсутствие поля ≠ отсутствие функции; сравнение на сайте вводит в заблуждение.

| Свойство | [Archer C6](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-5/archer-c6/) | [Archer AX55](https://tp-link.ru/catalog/routery-wi-fi/wi-fi-6/archer-ax55/) |
|----------|--------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| `Поддержка WPS` | есть | **нет в спеках** |
| `Межсетевой экран (Firewall)` | есть | **нет в спеках** |
| `Защита от атак Denial-of-service (DoS)` | нет | **нет** |
| `Стандарт WPA3` | есть | **нет в спеках** |
| `Гарантия` | 12 мес. | 48 мес. |

У AX55 как у более новой модели часть полей безопасности **пропущена в карточке**, хотя в маркетинговом тексте заявлен WPA3.

### Системные замечания по именованию свойств

| Паттерн | Примеры на сайте |
|---------|------------------|
| WiFi vs Wi‑Fi | свойство `Стандарт WiFi` vs заголовки «Wi‑Fi 6»; у MR100 — `Стандарт Wi-Fi 802.11n, 2.4 ГГц` |
| «Кол-во» vs «Количество» | `Кол-во портов USB 3.0` / `Количество выходных портов…` / `Количество антенн` |
| Разные форматы портов | `10/100/1000BASE-TX` (AX55) vs `100/1000/2500BASE-T` (AX80) vs `10/100BASE-TX/4G` (MR100) |
| Комплект поставки без разделителей | строка «Wi-Fi роутерАдаптер питанияКабель…» (BE230) или с пробелами (C6) — непоследовательный формат |
| Маркетинг внутри спек | свойство `Преимущества` (MR100) — многострочный рекламный текст вместо структурированного да/нет |

### Примечания по реализации

- Для фильтра выводить только свойства с реальной селективностью (убрать `DHCP_SUPPORT`, если везде `Да`).
- Все скорости/частоты/память хранить как числа; единицы добавлять на уровне шаблона.
- Для региональных версий (EU/RU) обязательны `COUNTRY_CODE` и уникальный составной ключ (`ARTICLE + COUNTRY_CODE`).
- Старые разрозненные поля (`Стандарт Wi-Fi 802.11b/g/n`) маппить в новую схему через слой миграции.
